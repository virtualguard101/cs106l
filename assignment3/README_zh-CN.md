## 第一部分：创建你的类

尽情发挥你的创造力吧！在 `class.h` 和 `class.cpp` 中填写代码，创建你自己的自定义类。更多信息请参考周二关于类的讲座中的相关幻灯片。你的类可以代表几乎任何你想要的东西，只要满足以下要求即可。

> [!IMPORTANT]
> 
> ### 类的要求
> 
> 你的类必须：
> 
> 1.  拥有一个自定义构造函数，接受**一个或多个**参数。
> 2.  拥有一个默认（无参数）构造函数（即构造函数重载）。
> 3.  拥有一个或多个私有成员字段（即变量）。
> 4.  拥有一个或多个私有成员函数。
>     *   请记住，私有函数就像汽车引擎盖下的运作机制！它们是类实现中不可或缺的部分，但不应暴露在公共接口中。试着思考一个在类上下文中逻辑合理的私有成员函数。
> 5.  拥有**至少一个**针对私有字段的公共 getter 函数。
>     *   例如，如果字段是 `int data`，则必须有一个名为 `get_data` 或 `getData` 的函数，其签名为
>         
>         int getData();
>         
>     *   getter 函数还应标记为 `const`。如果你不熟悉 `const` 的正确用法，请参考周四的课程内容！
> 6.  至少为其中一个私有字段提供一个公共的 setter 函数。
>     *   例如，如果字段是 `int data`，则必须有一个名为 `set_data` 或 `setData` 的函数，其签名为
>         
>         void setData(int value);
>         

请注意，这是完成作业的最低要求。如果你想要额外练习，完全可以超越这些要求，或者创建多个类！

> [!NOTE] 加分项：你可以选择使用周四讲座中讨论的 `template <typename T>` 符号创建类模板，而非普通类。这完全是可选的！
> 
> 请注意，如果你决定创建类模板， **必须从编译命令中移除 class.cpp。** 例如，在 Mac/Linux 上，编译命令应为：
> 
> ```sh
> g++ -std=c++20 main.cpp -o main
> ```
> 
> 同时记得调整包含关系，使 `.h` 文件在末尾包含 `.cpp` 文件，正如周四课程中所讨论的那样。

现在你已经创建了类，让我们实际使用它。 **在 `sandbox.cpp` 文件的 `sandbox` 函数内部，构造一个你创建的类的实例！** 你可以用任何你喜欢的方式来实现（调用默认构造函数、使用统一初始化等）。
