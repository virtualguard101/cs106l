## 运行你的代码

要运行你的代码，首先需要编译它。打开终端（如果你使用的是 VSCode，按 Ctrl+\` 或前往顶部菜单的 终端 > 新建终端）。然后确保你位于 `assignment2/` 目录下，并运行：

```sh
g++ -std=c++20 main.cpp -o main
```

假设你的代码编译时没有出现任何编译器错误，现在你可以执行：

```sh
./main
```

这将会实际运行 `main.cpp` 中的 `main` 函数。

在你按照以下说明操作时，我们建议你间歇性地使用自动评分器进行编译/测试，以确保你走在正确的轨道上！

## 第 1 部分：获取所有申请人

你等了好几天，终于收到了今年《婚姻契约》的姓名首字母匹配结果！今年他们实施了一项新规则：你的匹配对象必须与你拥有相同的姓名首字母才符合条件。然而，即使和朋友们讨论了好几个小时，你仍然猜不到自己的匹配对象是谁！校园里有成千上万的学生，你不可能手动翻遍整个名单来列出潜在灵魂伴侣的候选名单。幸运的是，你正在学习 CS106L 课程，还记得 C++有一种快速处理收集到的相似信息的方法——容器！

我们提供了一个包含今年所有（虚构）《婚姻契约》报名学生的 `.txt` 文件（ `students.txt` ）。每行包含学生的名和姓。首先，你需要编写函数 `get_applicants` ：

> [!IMPORTANT]
> 
> ### `get_applicants`
> 
> 从 `.txt` 文件中，将所有姓名解析到一个集合中。名为 `filename` 的文件中的每一行都是一个申请人的姓名。在实现过程中，你可以自由选择使用有序集合（ `std::set` ）或无序集合（ `std::unordered_set` ）！如果选择使用无序集合，请修改相关的函数定义！

另外，请在 `short_answer.txt` 中回答以下简答题：

> [!IMPORTANT]
> 
> ### `short_answer.txt`
> 
> Q1：你可以选择使用有序集合或无序集合。请用几句话说明两者之间的一些权衡。此外，请给出一个（课堂上未展示过的）可用于无序集合中哈希学生姓名的有效哈希函数示例。

> [!NOTE] 本作业中出现的所有姓名均为虚构。如有雷同，纯属巧合。

## 第二部分：寻找匹配对象

出色的侦探工作！既然你已经缩小了潜在灵魂伴侣的候选名单，是时候进行实战检验了。经过一整天无伴奏合唱和咨询社团会议的漫长时光，你回到宿舍，从室友那里得知今晚主广场正在举办一场婚姻契约匹配联谊会！你找到真爱的绝佳机会近在咫尺——前提是你能从极限飞盘训练中脱身。你当机立断，决定在联谊会上与所有和你姓名首字母相同的人交谈，并着手编写一个能自动生成会面顺序的函数。

在本部分中，你需要编写函数 `find_matches` 和 `get_match` ：

> [!IMPORTANT]
> 
> ### `find_matches`
> 
> 从集合 `students` （上一部分生成）中，取出所有与参数 `name` 首字母相同的名字，并将指向它们的指针放入新的 `std::queue` 中。
> 
> *   如果你在遍历集合时遇到困难，回顾一下周四关于迭代器和指针的讲座可能会有所帮助。
> *   完成这部分需要熟悉 `std::queue` 的操作。请查看 cppreference 的文档（[链接在此](https://zh.cppreference.com/w/cpp/container/queue)）。
> *   提示：定义一个辅助函数来计算某个学生名字的首字母可能会很有帮助。然后你可以用这个辅助函数将 `name` 的首字母与 `students` 中每个名字的首字母进行比较。

请从这里开始实现函数 `get_match` ，以找到你的“唯一真命匹配”：

> [!IMPORTANT]
> 
> ### `get_match`
> 
> 从所有可能匹配的队列中获取你的“唯一真命匹配”。你可以自行决定如何确定；选择某种方法从队列中获取一名学生，理想情况下，方法应比单一的 `pop()` 更有思考性，但不必特别复杂！可以考虑随机值或其他选择方法。
> 
> 如果你的首字母在数据集中没有匹配项，则打印 `“NO MATCHES FOUND.”` 明年好运 😢
